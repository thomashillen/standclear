# Code of Conduct

StandClear is a small open-source project. The expectation is simple: treat everyone in the project — contributors, maintainers, users opening issues — the way you'd want to be treated yourself.

## In practice

- **Be kind.** No personal attacks, harassment, slurs, or hostile behavior, in issues, PRs, commits, code review, or any other project space.
- **Assume good intent.** A confused question is a chance to teach, not to dunk.
- **Keep it on-topic.** Project spaces aren't for soapboxing; bring transit, software, and product feedback.
- **Disagreement is welcome; contempt isn't.** "I don't think this approach works because X" is great. "This is stupid" isn't.

## Reporting

If someone is making the project unsafe or unwelcoming for you or others, email the maintainer (see [About](https://standclear.app/about)) or open an issue. Reports stay confidential.

Maintainers may, at their discretion, edit or remove comments, close PRs, or ban contributors who repeatedly violate these expectations. The bar is "is this person making the project worse to be in for everyone else." If yes, they leave.

## Inspiration

This is intentionally short. If you want a more formal version, the [Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) is a good starting point and we'll defer to its spirit on anything not explicitly covered here.
