# Native build guide

This document is the canonical reference for building, testing, and shipping StandClear's native iOS (and later Android) app via Capacitor.

The native app is a thin Capacitor shell that loads the live web app at `https://standclear.app` inside a WebView, layered with native plugins (splash screen, status bar styling, share, preferences). This pattern lets us iterate on the web app instantly while still satisfying Apple's App Review guideline 4.2 ("apps must offer some level of native interaction").

## Prerequisites

| Requirement | iOS | Android |
| --- | --- | --- |
| OS for native build | macOS (any recent) | macOS, Linux, or Windows |
| IDE | Xcode 15+ | Android Studio Hedgehog+ |
| Developer account | Apple Developer ($99/yr) | Google Play Developer ($25 one-time) |
| Device for real-device testing | iPhone with Apple ID | Android phone with USB debugging |
| CocoaPods | Yes — `sudo gem install cocoapods` | n/a |

The Node toolchain is the same as for the web app; nothing extra needs to be installed in your shell beyond what `npm install` already pulled.

## Project layout

```
capacitor.config.ts        # Bundle ID, app name, server URL, plugin config
native-shell/              # Tiny offline-fallback HTML used as a webDir placeholder
ios/                       # Xcode project (committed) — generated by `cap add ios`
  App/
    App.xcodeproj/         # Open this in Xcode to build
    App.xcworkspace/       # ALWAYS open the workspace, not the project, after running pod install
    App/                   # Source: AppDelegate.swift, Info.plist, asset catalog
    Podfile                # CocoaPods manifest — `pod install` regenerates Pods/
components/NativeBoot.tsx  # Mounts in app/layout.tsx; hides splash + sets status bar
components/RegisterSW.tsx  # Skips service-worker registration when running in native
components/InstallPrompt.tsx # Skips A2HS prompt when running in native
lib/native.ts              # `isNative()` + Capacitor lifecycle helpers
```

## Daily development workflow

You'll mostly iterate on the web app exactly as before — `npm run dev`, edit, save, reload. The native app loads `https://standclear.app` in production, so once your web changes are merged and Vercel-deployed, the native app picks them up on the next launch with no rebuild required.

Native code only needs touching when you:

1. Add or update a Capacitor plugin (`@capacitor/push-notifications`, `@capacitor/haptics`, etc.).
2. Change `capacitor.config.ts`.
3. Change anything in `ios/App/`.

After any of those, run:

```bash
npm run cap:sync:ios    # Copy web shell + plugin updates into the iOS project
```

Then either rebuild from Xcode or:

```bash
npm run cap:open:ios    # Opens ios/App/App.xcworkspace in Xcode
```

## First-time iOS setup (run on your Mac)

1. Pull the branch with the iOS project committed.
2. `npm install` to fetch JS deps including the Capacitor plugins.
3. `cd ios/App && pod install && cd -` — this fetches the native iOS plugin pods.
4. `npm run cap:open:ios` — opens the Xcode workspace.
5. In Xcode: select the **App** target → **Signing & Capabilities** tab → set your **Team** to your Apple Developer account. The bundle identifier `app.standclear` is reserved by you the first time you upload to App Store Connect; until then, you can change it temporarily for development if needed.
6. Pick a simulator from the run-target dropdown (e.g. *iPhone 15 Pro*) and hit ⌘R. The app should boot to the dark splash, transition into the live `standclear.app` map.

## Testing on a real device

1. Plug your iPhone in via USB and trust the Mac when prompted on the phone.
2. In Xcode, select the device from the run-target dropdown.
3. ⌘R. First run will install a development provisioning profile; you may need to **Settings → General → VPN & Device Management → Developer App** on the phone to trust it.
4. Once installed, the StandClear icon appears on the home screen. From here, future Vercel deploys ship to it instantly with no rebuild.

## App Store submission checklist

When the app is ready to ship:

### Pre-submission

- [ ] Bump `version` in `package.json` (this also updates `lib/site.ts`'s `VERSION` because they share a source).
- [ ] In Xcode, bump **Build** number (CFBundleVersion) — must monotonically increase per upload, even within the same Marketing Version.
- [ ] Verify the iOS asset catalog (`ios/App/App/Assets.xcassets/AppIcon.appiconset`) has all required icon sizes. App Store Connect rejects uploads missing the 1024×1024 marketing icon.
- [ ] Verify `Info.plist` privacy strings are accurate. The committed defaults:
  - `NSLocationWhenInUseUsageDescription` — "StandClear uses your location to surface nearby subway stations and walking routes to the platform. Your location stays on this device — it is never sent to our servers." (populated)
  - `NSMotionUsageDescription` — "StandClear uses motion data only for the optional reactive-glass tilt effect on panels and pills. The data never leaves your device and the effect can be disabled in More → Personalize." (populated)
  - `NSAppTransportSecurity` — defaults to HTTPS-only; we don't load any plaintext domains. No exception keys set.
- [ ] Confirm `capacitor.config.ts`'s `server.url` points at production (`https://standclear.app`), not a preview deployment.
- [ ] Run the PR checks one more time: `npm run lint && npx tsc --noEmit && npm test`.

### App Store Connect setup (one-time)

1. Log in to https://appstoreconnect.apple.com.
2. Create a new app:
   - **Platform:** iOS
   - **Bundle ID:** `app.standclear` (must match `capacitor.config.ts`)
   - **SKU:** any unique string, e.g. `standclear-ios-1`
   - **Primary Language:** English (U.S.)
3. Fill out **App Information**:
   - **Category:** Navigation (primary), Travel (secondary)
   - **Content Rights:** Yes, I have the rights to use the MTA logo? No — we don't use it directly. Use the 🚇 emoji + "StandClear" wordmark only.
   - **Age Rating:** 4+
4. Fill out **Pricing and Availability**:
   - Free
   - Available in: United States (initially; expand later if traffic grows abroad)
5. Fill out **App Privacy**:
   - **Data Linked to You:** None
   - **Data Used to Track You:** None
   - **Data Not Linked to You:** Diagnostics (Vercel Analytics, anonymized)
   - Reference the live `/privacy` page URL in the App Privacy Policy field.

### The actual submission

1. In Xcode: **Product → Archive**. Wait for the archive to build (5–10 min on a fresh project).
2. The Organizer window opens; click **Distribute App** → **App Store Connect** → **Upload**.
3. Xcode validates and uploads to App Store Connect (5–15 min).
4. In App Store Connect: under **TestFlight**, the build appears with a "Processing" badge. Wait 5–30 min for processing.
5. Once processed, **TestFlight → Internal Testing** lets you install on your own devices via the TestFlight app for final QA.
6. Move to **App Review** when ready: select the build in the version's "Build" section, fill in:
   - **App Description** — copy from the `/about` page; tighten to ~3 paragraphs.
   - **Keywords** — `nyc subway, mta, real-time subway, transit, subway tracker, live arrivals` (100 char limit, comma-separated).
   - **Screenshots** — capture from a real iPhone, in light + dark mode if relevant. Required sizes:
     - 6.7" iPhone (iPhone 15 Pro Max): 1290×2796 px
     - 6.5" iPhone (iPhone 11 Pro Max): 1242×2688 px
     - 5.5" iPhone (iPhone 8 Plus): 1242×2208 px
   - **App Review Information** — provide a demo account if you ever add accounts (currently N/A; the app has no login).
   - **Notes for Reviewer** — *"StandClear is a free, ad-free real-time NYC subway tracker. The app uses MTA's public GTFS-Realtime feeds. Native value-add: splash screen, status bar styling, share extension, native preferences storage, native lifecycle integration. Source: https://github.com/thomashillen/standclear (MIT). Privacy: https://standclear.app/privacy"*
7. Click **Submit for Review**.

### Common rejection reasons + fixes

- **Guideline 4.2 (Minimum Functionality)** — "Your app does not provide a minimum amount of native functionality." Fix: emphasize the native plugins we ship (splash, status bar, share, preferences) in the reviewer notes; demonstrate them in a TestFlight build.
- **Guideline 5.2.5 (Apps & Metadata that incorporate third-party trademarks)** — if the reviewer thinks we're using the MTA's branding. Fix: confirm we use only the 🚇 emoji and "StandClear" wordmark, never the MTA bullet logos directly. The route bullets in the UI are colored circles with letter labels — this is acceptable as functional UI, but if challenged, point at the MTA developer terms which permit it.
- **Guideline 2.1 (App Completeness)** — broken links, blank pages, missing privacy policy. Fix: walk through every panel before submitting; verify `/privacy` and `/terms` are reachable.

## Updating the web app post-launch

Web changes ship instantly to the native app — no resubmission needed. Just merge to `main`, let Vercel deploy, and users get the new version on the next app launch.

Native changes (capacitor.config.ts, Info.plist, native plugin updates) require a new App Store build:

1. Edit + commit on a branch.
2. Bump `package.json` version + Xcode build number.
3. Archive + upload as above.

## Adding Android later

When you're ready:

1. `npm install --save @capacitor/android` (already pre-emptive in `package.json`? if not, add it).
2. `npm run cap:add:android` — generates `android/`.
3. Open `android/` in Android Studio.
4. Sign with a release keystore.
5. Submit to Google Play Console.

The Capacitor config in `capacitor.config.ts` already covers Android-specific keys (`androidScheme`, `SplashScreen.androidScaleType`); adding the platform should be largely additive.

## Troubleshooting

**"Pod install fails with `xcrun: error: invalid active developer path`"**
Run `xcode-select --install` to install the command line tools, then retry.

**"App boots to a white screen on simulator"**
The simulator can't resolve `https://standclear.app` if it's offline or DNS is broken. Check the simulator's network. The fallback `native-shell/index.html` should appear if the live site is unreachable; if even that doesn't show, check Xcode's debug console for WebView load errors.

**"Splash screen never hides"**
`NativeBoot.tsx` calls `SplashScreen.hide()` after hydration. If hydration is failing (check the Xcode console for JS errors), the splash will stay until `launchShowDuration` (1.2s) elapses, then auto-hide via `launchAutoHide: true` in capacitor.config.ts. So worst case the splash hangs ~1.2s. If it hangs longer, hydration is broken.

**"Status bar overlaps the floating header"**
The header has a `paddingTop: calc(max(var(--safe-top), 0.5rem) + 0.5rem)` rule. Verify the app's WebView is reading `safe-area-inset-top` — Capacitor injects this via the Info.plist `UIRequiresFullScreen=NO` setting, which should be the default.

**"App Review rejected for using third-party data without authorization"**
The MTA explicitly permits use of GTFS-Realtime under their developer terms — link to https://api.mta.info/ in the rebuttal.
